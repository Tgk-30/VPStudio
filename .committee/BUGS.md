# VPStudio bug list

Living list of bugs, regressions, UX issues, and investigation targets.

## QA / verification standard

### Release gate
- Bump VPStudio to **version 1.1** only after the tracked changes are successful, working, and verified with QA evidence.
- Do not bump the version early just because code landed; treat **1.1** as the post-fix / post-verification release target.

Every fix in this list should be verified with explicit QA before it is considered done.

### Required verification for each fix
- Run QA against the affected flow/screen after the change
- Capture screenshots of the affected UI states
- Carefully analyze the screenshots instead of only checking whether the app compiles or superficially "looks fine"
- Confirm the fix actually solved the problem without introducing obvious visual regressions
- Do not mark an item as fixed until QA evidence has been reviewed

### Screenshot review expectations
- Check layout, sizing, spacing, proportions, and readability
- Check loading/empty/error states where relevant
- Check tap targets and usability, especially for Vision Pro controls
- For player/window/layout fixes, verify multiple sizes/states if applicable
- Prefer evidence-based verification over assumption

## Suspects / technical notes

These are not confirmed bugs by themselves, but they may contribute to the issues above.

### A) Stream failover may be too naive
